= Installing Cassandra

[IMPORTANT]
====
*Preview | Unofficial | For review only*
====

Before installing, choose your deployment model.
The right installation method depends on whether you are running a local evaluation, a production pilot, or a production cluster.
See xref:deployment-guide.adoc[Choose Your Deployment Model] for a decision guide covering bare metal, cloud IaaS, and Kubernetes.

For Kubernetes deployments, installation is managed by the K8ssandra operator rather than the methods below.
See xref:automate/kubernetes-considerations.adoc[Kubernetes Considerations] for guidance.

== Supported Platforms

Apache Cassandra runs on the following Linux distributions:

* AlmaLinux
* Amazon Linux AMIs
* Debian
* RedHat Enterprise Linux (RHEL)
* SUSE Enterprise Linux
* Ubuntu

This is not an exhaustive list.
Test thoroughly on less common distributions before production use.
Deploying on older Linux versions is not recommended unless you have direct experience with that distribution in production.

== Prerequisites

=== Java Runtime

Cassandra 6 requires **JDK 17 minimum**; **JDK 21 is strongly recommended** for production deployments.
JDK 21 enables Generational ZGC, which delivers the best garbage collection behavior for Cassandra workloads.

[cols="1,1,2",options="header"]
|===
|JDK Version |Support Status |Notes

|JDK 21 (LTS)
|**Recommended**
|Required for Generational ZGC. Best GC performance for production.

|JDK 17 (LTS)
|Supported
|Minimum supported version for Cassandra 6. Use JDK 21 when possible.

|JDK 11
|Not supported in Cassandra 6
|Supported in Cassandra 4.x and 5.x only.
|===

Download JDK 21 from one of the following sources:

* https://adoptium.net/temurin/releases/?version=21[Eclipse Temurin (OpenJDK 21)] — recommended open-source distribution
* https://www.oracle.com/java/technologies/downloads/#java21[Oracle JDK 21]
* https://jdk.java.net/21/[OpenJDK 21 (jdk.java.net)]

include::Cassandra:cassandra:partial$java_version.adoc[]

=== Python (for cqlsh)

To use the CQL shell `cqlsh`, install Python 3.8–3.11.

[source,bash]
----
python --version
----

[NOTE]
====
Python 2.7 support is removed. Python 3.8 or later is required.
====

== Choosing an Installation Method

[cols="1,2,2",options="header"]
|===
|Method |Best For |Notes

|**Package (RPM/DEB)**
|Production installs on supported Linux distributions
|Requires `root`. Installs binaries as the `cassandra` OS user. Managed by APT or YUM.

|**Tarball**
|Development, testing, and installs without root
|No root required. Works on any Linux distribution. Recommended for contributors and lab environments.

|**Docker**
|Local evaluation and development only
|Not for production. Useful for getting started quickly or running integration tests locally.
|===

[WARNING]
====
The Docker method is suitable for local evaluation and development.
Do not use it for production clusters.
For production, use package installs on supported Linux distributions.
====

[WARNING]
====
Note that both the YUM and APT methods required `root` permissions and will install the binaries and configuration files as the `cassandra` OS user.
====

== Install with Docker

. Pull the docker image. For the latest image, use:
+
[source, shell]
----
include::Cassandra:cassandra:example$BASH/docker_pull.sh[]
----
+
This `docker pull` command will get the latest version of the official Docker
Apache Cassandra image available from the https://hub.docker.com/_/cassandra[Dockerhub].
. Start Cassandra with a `docker run` command:
+
[source, shell]
----
include::Cassandra:cassandra:example$BASH/docker_run.sh[]
----
+
The `--name` option will be the name of the Cassandra cluster created.
This example uses the name `cass_cluster`.
. Start the CQL shell, `cqlsh` to interact with the Cassandra node created:
+
[source, shell]
----
include::Cassandra:cassandra:example$BASH/docker_cqlsh.sh[]
----
