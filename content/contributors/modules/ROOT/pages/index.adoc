= Contributing to Cassandra

[IMPORTANT]
====
*Preview | Unofficial | For review only*
====

Apache Cassandra is an open-source project that welcomes contributions of all kinds: code, documentation, tests, bug reports, and tooling improvements.
Whether you are fixing your first bug or implementing a major feature, this section provides everything you need to contribute effectively.

== Find Your Path

Not sure where to start?
The xref:orientation/roadmap.adoc[Contributor Roadmap] helps you choose the right path based on what you want to do: fix a bug, add a feature, learn the codebase, write tests, improve documentation, or review patches.

== Learn the Codebase

Cassandra is a large distributed systems codebase.
These resources make it legible:

* xref:internals/internals-map.adoc[Cassandra Internals Map] -- every major subsystem mapped to source packages, key classes, and cross-subsystem boundaries
* xref:internals/query-path.adoc[Query Execution Path] -- trace a CQL query from client to disk and back
* xref:internals/messaging.adoc[Messaging and Failure Handling] -- how nodes communicate and detect failures
* xref:sstable-architecture/fundamentals.adoc[SSTable Architecture] -- deep dive into the storage engine, from the SSTable book
