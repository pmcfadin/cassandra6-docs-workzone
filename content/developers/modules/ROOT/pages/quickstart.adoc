= Developer Quickstart

Get up and running with Apache Cassandra in minutes.
Choose your path: explore CQL interactively or build your first application in your preferred language.

== Start Cassandra

The fastest way to get a development instance running:

[source,bash]
----
docker pull cassandra:latest
docker run --name cassandra -d -p 9042:9042 cassandra:latest
----

Wait about 30 seconds for the node to initialize.
Verify it's ready:

[source,bash]
----
docker exec cassandra cqlsh -e "DESCRIBE KEYSPACES"
----

For detailed setup options, see the xref:Cassandra:cassandra:getting-started/cassandra-quickstart.adoc[Cassandra Quickstart].

=== Explore CQL First

If you prefer to learn the query language before writing application code, connect with cqlsh:

[source,bash]
----
docker exec -it cassandra cqlsh
----

==== Create a Keyspace

[source,sql]
----
CREATE KEYSPACE my_app
  WITH replication = {'class': 'SimpleStrategy', 'replication_factor': 1};

USE my_app;
----

==== Create a Table

[source,sql]
----
CREATE TABLE users (
  user_id uuid PRIMARY KEY,
  name text,
  email text,
  created_at timestamp
);
----

==== Insert Data

[source,sql]
----
INSERT INTO users (user_id, name, email, created_at)
  VALUES (uuid(), 'Alice', 'alice@example.com', toTimestamp(now()));

INSERT INTO users (user_id, name, email, created_at)
  VALUES (uuid(), 'Bob', 'bob@example.com', toTimestamp(now()));
----

==== Query Data

[source,sql]
----
SELECT * FROM users;
----
