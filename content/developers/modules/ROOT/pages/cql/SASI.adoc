= SASI Indexes

[IMPORTANT]
====
SASI is a legacy index implementation and is no longer recommended in the developer docs.
Use xref:cql/indexing/sai/sai-concepts.adoc[Storage-Attached Indexing (SAI)] for new development.
====

This page remains only as a migration pointer for existing links and older deployments.

== What To Do Instead

For new indexing work:

* Use xref:guides/sai-usage-patterns.adoc[SAI Usage Patterns] for developer guidance
* Use xref:cql/indexing/sai/sai-concepts.adoc[SAI Concepts] for architecture and behavior
* Use xref:vector-search/index.adoc[Vector Search] if you need ANN queries on `VECTOR` columns

== Migrating Existing SASI Usage

If your application still depends on SASI:

. Identify the query paths that rely on the SASI index.
. Create the equivalent SAI index for the same access pattern.
. Wait for the SAI index to finish building and verify query behavior.
. Remove the SASI index after the replacement path is validated.

See xref:guides/sai-usage-patterns.adoc#migrating-from-sasi-to-sai[Migrating from SASI to SAI] for practical migration guidance.
