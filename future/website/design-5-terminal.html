// cassandra --features

feature overview
$ cassandra --features
DISTRIBUTED
No single point of failure. Peer-to-peer masterless architecture scales horizontally without coordination overhead.
FAULT_TOLERANT
Multi-datacenter replication. Continuous availability with tunable consistency — survive datacenter failures automatically.
PERFORMANT
Sub-millisecond reads. Log-structured merge tree storage delivers predictable low-latency writes at any scale.
SCALABLE
Add nodes with zero downtime. Linear throughput scaling — double nodes, double capacity, no re-sharding required.
ELASTIC
Zero Copy Streaming. Instant cluster expansion with v6 improvements. Rebalance without full data movement.
SECURE
Audit logging. Role-based access control. TLS encryption at rest and in transit. CEP-security compliance.
VECTOR_SEARCH
Native SAI vector indexing. ANN queries over high-dimensional embeddings with no external index infrastructure required.

// cassandra query language

SQL-familiar.
Distributed-native.

CQL is the query language of Apache Cassandra. It looks and feels like SQL, but it's designed ground-up for distributed, partitioned data. No joins required — just intentional data modeling.

Cassandra 6 introduces new SAI vector search, improved secondary index performance, and a refreshed WASM-based UDF engine — all accessible through the same familiar CQL interface.

CQL sensor_readings.cql
-- Create a keyspace with replication CREATE KEYSPACE iot WITH replication = { 'class': 'NetworkTopologyStrategy', 'dc1': 3 }; -- Time-series table for IoT sensor data CREATE TABLE iot.readings ( device_id UUID, recorded_at TIMESTAMP, sensor TEXT, value DOUBLE, tags MAP<TEXT, TEXT>, PRIMARY KEY ((device_id), recorded_at) ) WITH CLUSTERING ORDER BY (recorded_at DESC); -- Vector search index (Cassandra 6) CREATE CUSTOM INDEX reading_vec_idx ON iot.readings (embedding) USING 'StorageAttachedIndex' WITH OPTIONS = {'similarity_function': 'cosine'}; -- ANN vector query SELECT device_id, recorded_at, value FROM iot.readings ORDER BY embedding ANN OF [0.1, 0.8, 0.3] LIMIT 10;

// community Apache Cassandra is a community project

There is no single company behind Cassandra. It is governed by volunteer committers and PMC members under the Apache Software Foundation. All decisions happen in the open — on mailing lists, in JIRA, and in code review.

New contributors are welcome at every level: documentation fixes, bug reports, feature proposals, and code patches. Start by joining the dev@ mailing list and picking a good-first-issue from JIRA.

Apache Cassandra is an Apache Software Foundation top-level project. The PMC and committer list, voting process, and release policy are all publicly documented. The ASF provides infrastructure, legal, and brand protection.
