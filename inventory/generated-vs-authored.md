# Generated Vs Authored

Capture date: **2026-03-24**

## Rule
Treat generated documentation and hand-authored documentation as separate workstreams. Generated outputs must be regenerated and validated before prose review or AI drafting.

## Hand-Authored Surfaces
- Most content under `doc/modules/ROOT` and `doc/modules/cassandra/pages/` is maintained as hand-authored AsciiDoc. Sources: [`apache/cassandra/doc`](https://github.com/apache/cassandra/tree/trunk/doc), [`doc/modules/cassandra/pages`](https://github.com/apache/cassandra/tree/trunk/doc/modules/cassandra/pages).
- Website-only content such as contributor guidance and top-level pages lives in `apache/cassandra-website/site-content/source/modules/ROOT/pages`. Source: [`site-content/source/modules/ROOT/pages`](https://github.com/apache/cassandra-website/tree/trunk/site-content/source/modules/ROOT/pages).

## Generated Surfaces
- `cass_yaml_file.adoc` is generated from `conf/cassandra.yaml` by `doc/scripts/convert_yaml_to_adoc.py`. Sources: [`doc/Makefile`](https://github.com/apache/cassandra/blob/trunk/doc/Makefile), [`doc/scripts/convert_yaml_to_adoc.py`](https://github.com/apache/cassandra/blob/trunk/doc/scripts/convert_yaml_to_adoc.py), [`trunk/.gitignore`](https://github.com/apache/cassandra/blob/trunk/.gitignore).
- `managing/tools/nodetool/` pages are generated by `doc/scripts/gen-nodetool-docs.py` and ignored in git. Sources: [`doc/scripts/gen-nodetool-docs.py`](https://github.com/apache/cassandra/blob/trunk/doc/scripts/gen-nodetool-docs.py), [`trunk/.gitignore`](https://github.com/apache/cassandra/blob/trunk/.gitignore).
- `reference/native-protocol.adoc` and related protocol artifacts are generated through `process-native-protocol-specs-in-docker.sh` and ignored in git. Sources: [`doc/Makefile`](https://github.com/apache/cassandra/blob/trunk/doc/Makefile), [`doc/scripts/process-native-protocol-specs-in-docker.sh`](https://github.com/apache/cassandra/blob/trunk/doc/scripts/process-native-protocol-specs-in-docker.sh), [`trunk/.gitignore`](https://github.com/apache/cassandra/blob/trunk/.gitignore).

## Planning Implications
- Do not ask an LLM to describe config defaults, nodetool behavior, or protocol details until the generated-doc step has been rerun for the target branch.
- Review generated outputs as evidence, not as prose to be freely rewritten.
- Track generated outputs explicitly in `inventory/docs-map.csv` so they are never omitted from a Cassandra 6 doc review.

## Working Heuristic
- If a page is derived from code, config, CLI help, or protocol specs, classify it as `generated`.
- If a page is maintained directly in `.adoc` without a generation script in the workflow, classify it as `authored`.
- If uncertain, treat it as authored plus evidence-required until generation provenance is confirmed.
